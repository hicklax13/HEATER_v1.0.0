## Correctness findings
- **LO-C1** (L4, **Medium**, conf 82) — *Counting stats discounted TWICE for availability.* `src/optimizer/projections.py:606-684` (Step 5 injury Weibull, scales counting cats by `expected_availability`) AND `:694-764` (Step 5b playing-time, scales SAME cats by `predicted_pa/proj_remaining_pa`). Both invoked from `build_enhanced_projections` (`:143-147`, enable_injury=True + enable_playing_time default True) in standard/full/quick. Both model the same latent (how much the player plays) and multiply independently → injury-prone + low-depth player gets e.g. 0.80×0.75=0.60, double-penalized. The PT model's PA already reflects injury-shortened seasons. **Fix:** model availability once (gate Step 5 off when PT prediction exists, or blend via min/geomean, or Step 5 adjusts confidence only). NOT a Known Design Choice (the CLAUDE.md PT-gate entries are about fa_recommender, separately audited). *Only finding with real run-time impact.*
- **LO-C2** (L4, **Low**, conf 78) — Pitcher park-factor inconsistent across paths. `matchup_adjustments.py:424-427` (daily DCV path: pitchers get reciprocal `1/pf`, Coors 1.38→0.72×) vs `:743-748` (weekly pipeline path: pitchers get NO park factor). Same pitcher ranked differently in Daily tab vs Optimize tab. Not a double-count (mutually exclusive per mode) but the two modes disagree. Pick one convention.
- **LO-C3** (L4, **Low**, conf 88) — Post-LP IP header card: stale comment claims RP IP ×4 weekly, but SP and RP branches both compute `proj_ip/26` (identical). `pages/2_Line-up_Optimizer.py:1474-1481`. CODE is correct (seasonal IP/26 already encodes reliever frequency); comment is wrong + dead branch. Display-only (header card), no LP/DCV impact.
- **LO-C4** (L4, **Low**, conf 70) — Streaming value for a 2-start week ignores the 0.93× second-start fatigue. `streaming.py:118-219` (`compute_streaming_value` ×weekly_games at un-fatigued ERA/WHIP) vs `:383-404` (`quantify_two_start_value` applies era/0.93). A 2-start streamer's ERA/WHIP damage understated vs the engine's own two-start convention.
- **LO-C5** (L4, **Low**, conf 55) — LP objective normalizes by within-roster stat stdev (`_compute_scale_factors`, lineup_optimizer.py:171,629-717) rather than league SGP denominators → category priority is roster-relative not standings-relative (low-variance cats spuriously up-weighted). category_weights layer on top, but base scale is roster-internal dispersion. Deviates from the SGP-denominator convention used by DCV + trade engine. Low conf (documented as deliberate; structural tests pass).

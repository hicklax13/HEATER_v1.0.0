# B6 — Draft engine (subagent result)

**Verdict:** Snake-order math verified empirically CORRECT (draft_state, simulation, pick_predictor all agree: round%2==0 forward / odd reverse, 0-based round; 4-team/4-round user picks 1,8,9,16). Draft valuation routes through canonical SGPCalculator/value_all_players (VORP graduated scarcity, marginal_sgp correct inverse signs, AI-opponent weighting uses _signed_magnitude_clip preserving negative-SGP sign). No parity/inversion/SGP-reinvention bug. Findings are MC seed discipline + a few latent/dead-path issues. (Preseason tool — in-season now, so lower live priority.)

## Correctness findings
- **DE-C1** (L4, **Medium**, conf 95) — *MC draft sim has no seed discipline / no common-random-numbers across candidates → #1 recommendation non-reproducible and noise-dominated at the margin.* `src/simulation.py:351` `rng = np.random.default_rng()` created fresh + UNSEEDED inside every `simulate_draft()`; `evaluate_candidates` (`:631-658`) calls it once per candidate → each candidate evaluated against INDEPENDENT random opponent draws. **Auditor reproduced live:** with 120-200 pool + n_sims=300, the #1 pick flips between two near-tied players (39.279 vs 39.298 — gap < MC noise) across identical re-runs. No seed plumbed. Violates the same "Paired MC seed discipline / antithetic variates" invariant the trade engine enforces. NOT a Known Design Choice. *This is the surface the user reads for close calls — exactly where noise matters most.*
- **DE-C2** (L4, **Medium**, conf 88) — `per_category_sgp` re-alignment in `evaluate_candidates` built in POOL order (`simulation.py:610-621`, `avail_indices = [i for i,pid in enumerate(pool_ids) if pid in sim_ids_set]`) but consumed in `sim_available`/ADP order (`:593-600`, `:366-371`) → category-aware user picks read the WRONG player's per-category SGP. Length guard passes, masking it. Fix: reindex by player_id like the volatility path already does (`:604-606`). **Latent** — no production caller passes category_weights+per_category_sgp (recommend() omits them; per_category_sgp "does not currently exist, must be built" per archived spec). Bites whoever wires category-aware draft MC.
- **DE-C3** (L4, **Medium**, conf 85) — `enhanced_pick_score` floors negative `base_sgp` to +0.01 (`draft_engine.py:1157-1180`), collapsing all below-replacement players to one value; additive bonuses (closer_hierarchy up to +2.0, lineup_protection +0.3) then invert order — a deeply-negative-VORP closer scores ~+2.0 and outranks a slightly-negative everyday hitter. Re-introduces the D6B-022 sign-loss class the sim itself fixed (`simulation.py:18-40` `_signed_magnitude_clip`). Use signed-magnitude clip. Mostly late-round/bench impact → medium.
- **DE-C4** (L5, **Low**, conf 80) — `pick_predictor._future_user_picks` (`:49-73`) includes the user's CURRENT on-the-clock pick as a "future" survival point (off-by-one; picks_between=1 for the current pick). Snake parity otherwise correct. **Dead in prod** (compute_survival_curve/_future_user_picks called only in tests).
- **DE-C5** (L5, **Low**, conf 92) — `test_horizon_limit` asserts `num_teams*6` but code uses `num_teams*4` (`simulation.py:388`); the test recomputes its own arithmetic and NEVER calls simulate_draft → horizon contract unverified + docstring wrong (4 vs 6 rounds). The user's projected roster value is a 4-round greedy lookahead.
- **DE-C6** (L4, **Low**, conf 90) — `draft_analytics.py` docstring claims it's wired into `DraftRecommendationEngine.enhance_player_pool()`, but `draft_engine.py` imports/calls NONE of its functions (reimplements category balance + BUY/FAIR/AVOID internally). The 4 analytics fns are effectively dead in the draft path (only power_rankings uses one). Two divergent implementations = drift risk. Fix docstring or wire to canonical.

## Enhancements (ranked)
1. **DE-E1** (High/Low) — Common-random-numbers + optional seed for candidate MC comparison (fixes DE-C1). Build one `rng=default_rng(seed)` in evaluate_candidates and share across candidates (or pre-draw a shared n_sims×horizon uniform tensor); add antithetic pairing for ~25% variance reduction. Test: two seeded calls return identical ordering AND candidate order doesn't change scores. Eliminates rank inversions for near-tied candidates + deterministic output.
2. **DE-E2** (Medium/Low) — Preserve sign of base value in enhanced_pick_score via `_signed_magnitude_clip` (fixes DE-C3). Restores below-replacement ordering; prevents +2.0 closer bonus elevating a worse player.
3. **DE-E3** (High/Medium) — Tier-aware / VOND value-cliff signal in recommendations (positional tier drop-off). Engine has assign_tiers() + per-position urgency but doesn't surface "last player before a tier cliff." Add next_tier_dropoff + players_left_in_tier columns + a "TIER CLIFF" flag; reuse compute_opportunity_cost (wire DE-C6). Modern dominant draft strategy; turns BPA into actionable reach-now-vs-wait.
4. **DE-E4** (Medium/Medium) — Thread the already-built per-player ADP sigma (`compute_player_adp_sigma`) + opponent need/history modeling into `recommend()` (currently uses a single global sigma=10.0, no team_preferences). More realistic survival/value, esp. for volatile players.
5. **DE-E5** (Medium/Medium) — Make draft_grader's expected-SGP baseline less self-referential (use replacement/positional baseline, not the pool's own sorted SGP-by-ADP). More defensible steal/reach grades.
